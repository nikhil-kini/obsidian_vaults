The word SOLID acronym for:

- Single Responsibility Principle (SRP) -  that every Java class must perform a single functionality.
- Open-Closed Principle (OCP) - the module should be open for extension but closed for modification.
- Liskov Substitution Principle (LSP) - design your classes so that client dependencies can be substituted with subclasses without the client knowing about the change.
- Interface Segregation Principle (ISP) - the larger interfaces are split into smaller ones. Because the implementation classes use only the methods that are required, we should not force the client to use the techniques that they do not want to use.
- Dependency Inversion Principle (DIP) - use abstraction (abstract classes and interfaces) instead of concrete implementations.
