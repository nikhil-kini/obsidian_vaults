## Inventory file
- Save IP address / Domain Name of the servers where ansible needs to run scripts, to `inventory`
```
172.16.250.132
172.16.250.133
172.16.250.134
```

Ping-pong test to connect to the server.
```sh
ansible all --key-file path_to_SHH_Scret_Key -i ip_file_path -m ping
```

## Ansible config

- Create a `ansible.cfg` file in local project to avoid writing SSH Path and IP file path for each command.
- **Note: No space and No comment # in file has its a`.cfg` file**
```
[defaults]
inventory = inventory                  # ip_file_path
private_key_file = ~/.ssh/ansible      # path_to_SHH_Scret_Key
```

- Shortend for ping pong command with ansible config
```sh
ansible all -m ping
```

- List all hosts
```sh
ansible all --list-hosts
```

- To get details on hosts, list all details about the hosts
```sh
ansible all -m gather_facts
```

- To get details on single host, list all details about the host
```sh
ansible all -m gather_facts --limit ip_address
```

### Tell ansible to use sudo

- To update the system
```sh
ansible all -m apt -a update_cache=true
```
will fail due to `sudo` permission issue.

To fix
```sh
ansible all -m apt -a update_cache=true --become --ask-become-pass
```

### Install a package via the apt module

```sh
ansible all -m apt -a name=vim-nox --become --ask-become-pass
```

### Install a package via the apt module, and also make sure it’s the latest version available or Update the existing package
```sh
ansible all -m apt -a "name=snapd state=latest" --become --ask-become-pass
```


### Upgrade all the package updates that are available
```sh
ansible all -m apt -a upgrade=dist --become --ask-become-passxx
```
